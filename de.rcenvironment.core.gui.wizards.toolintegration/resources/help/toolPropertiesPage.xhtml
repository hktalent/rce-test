Tool Integration Wizard

 

Tool Properties

Synopsis

Define properties for the tool, which can be used as parameters later in all scripts and can be configured when a workflow is set up.

Usage

Properties are a means to let the user of the integrated tool configure some attributes when creating a workflow. For example, if your tool is able to run with multiple cores, you can define a key “numCores” and later add it as placeholder in your command execution script or pre/post execution script (e.g. “tool.exe ${prop:numCores}”). This key will then be shown in the properties tab in the workflow editor and the value for it can be defined there.

Properties have a display name that makes them more readable for others. Also, a default value can be defined, to provide some useful presettings.

It is not mandatory to define properties.

Groups

With groups you can categorize the properties and group properties for the same context. It is also possible to create a file for each group at runtime, which contains the keys and the respective values. These files will be located in the “Config” folder in a temporary directory with the name that you can also define here.
