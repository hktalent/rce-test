Input Provider Component Usage

Synopsis

The Input Provider component sends specified values to inputs of components.

Rationale

The Input Provider sends specified values to others component, e.g. starting values. Therefore, the Input Provider writes defined values to its outputs. The outputs must be connected to the inputs of appropriate components. The values are sent once and immediately after the workflow is started. All of the data types are supported, which are available within RCE.

Usage

For each value, which should be sent, you need to create an output within the Input Provider component by clicking the "Add" button next to the outputs table. For each output you can decide whether to define the value directly or to define it at workflow start. Select the check box "Define at workflow start" appropriately.

If it is defined directly, it is stored in the workflow file. If you share the workflow with others, the defined values will be shared as well.

If it is defined at workflow start, only a placeholder is stored in the workflow file. It will be replaced with the value defined at workflow start. If you share the workflow with others, only the placeholder will be shared and the other person needs to define the value at workflow start by himself.

If you like to send files or directories to other components, you have three options:

1)      1) Select the file/directory to send at workflow start (you will ask for selection in the workflow start wizard). Choose that option, if you like to share the workflow (.wf file) with others and don't want to share the file/directory (e.g. because it contains sensitive data).

2)      2) Select the file/directory from the project in the workspace, where the workflow file is stored. Choose that option, if you like to share the workflow (.wf file) as well as the file/directory with others. In that case you need to share the whole project (e.g. as an archive file - right-click on the project and export it as an archive file). The other person needs to import that project into the RCE workspace. The workflow will run out-of-box using the file/directory of the shared project.

3)      3) Select the file/directory from file system. Choose that options if you don't like to share the workflow with others at all and if the file/directory needs to remain at a specified place in the file system.

You will see your outputs in the table similar like this.

After that, you can connect the outputs from the Input Provider to any other compatible inputs from other components. The values (either defined directly or defined at workflow start) will be sent to the connected inputs of the other components immediately after the workflow is started.
